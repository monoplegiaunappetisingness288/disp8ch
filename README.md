## 🛠 What this software does

Most AI tools hide the process inside a black box. disp8ch works differently. It shows you the steps. Each chat reply becomes a node in a workflow. You drag these nodes to change how the system acts.

This tool manages multiple agents at the same time. You can assign one agent to write code and another to check for errors. The system connects these agents so they cooperate. Your history remains on your drive. You gain full control over your automations.

## ⚙️ System requirements

Your computer needs specific hardware to run these models. Check these points before you start:

- Windows 10 or Windows 11 operating system.
- 16 gigabytes of RAM or more.
- An NVIDIA graphics card with at least 8 gigabytes of video memory.
- 10 gigabytes of free disk space for the program and models.
- A stable internet connection for the first setup.

These requirements ensure the AI responds without delay. If your computer lacks this hardware, the system may run slowly or struggle to process large tasks.

## 🚀 Setting up your first workflow

Upon opening the app, you see a blank canvas. This area holds your visual tasks. 

1. Click the chat bar at the bottom of the window.
2. Type a simple task, such as "List the files in my documents folder."
3. Press enter to send your request.
4. The system creates a node on your screen showing the intent.
5. Click the "Add Agent" button to assign a role to this task.
6. Select an AI model from the drop-down menu. 
7. Click "Run" to process the instruction.

The system will connect the nodes with lines. These lines show the flow of information. If the agent needs to move a file, you see the instruction pass through the line. You can add more nodes to create complex chains of action.

## 🧩 Understanding the components

The interface consists of four main sections.

**The Canvas**
This area displays your workflows. Use your mouse to drag items around. Scroll to zoom in or out.

**The Sidebar**
This list contains your history. You can find past workflows here to edit or run them again.

**The Model Selector**
This button sits at the top right of the application. It lets you switch between models for specific tasks.

**The Action Log**
This screen shows the internal steps of your agents. If a task reaches a wall, read this log to find the error. It explains why an agent stopped.

## 🔧 Troubleshooting common problems

If the program fails to start, verify that your graphics drivers are current. Outdated drivers cause errors during the GPU handshake.

- **The app shows an error during startup:** Close the app and restart your computer to clear the memory.
- **The screen displays a blank window:** Check your internet connection. The app may need to verify the model integrity.
- **The agent creates an error message:** Click the individual node to see the details of the request. Often, a missing file path causes this. Fix the path and click "Retry."
- **The AI behaves in unexpected ways:** Open the chat and ask the agent "Why did you pick this path?" The agent clarifies the logic behind its choice.

## 📜 Using agents effectively

Agents perform better with clear instructions. Instead of saying "Sort my emails," try "Find all emails from my bank and save the attachments to the Finance folder." Specificity allows the agent to build a more accurate chain of nodes. Use individual nodes for distinct parts of a task to keep your workflows organized. This makes it easy to spot the location of bugs if something goes wrong.
